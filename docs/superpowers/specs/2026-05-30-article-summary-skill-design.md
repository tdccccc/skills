# 天文论文总结 Skill — 设计文档

日期：2026-05-30
状态：已与用户确认，待实现

## 1. 目的

构建一个 Claude Code skill，把天文/学术论文转成中文结构化总结。用户给出论文来源（arXiv、PDF、LaTeX、DOI 等），skill 抓取内容、按固定模板总结、在对话中展示，并可按需存档。

## 2. 已确认的需求

| 维度 | 决定 |
|------|------|
| 定位 | 天文 / 学术论文专用 |
| 输入来源 | arXiv ID/链接、本地 PDF、本地 LaTeX/文本、DOI/期刊/ADS（四类全支持） |
| 输出内容 | TL;DR+核心发现 / 结构化拆解 / 公式与图表解读 / 局限+未来+相关工作（四块全要） |
| 语言 | 中文为主，专业术语保留英文 |
| 存档 | 默认只在对话中显示；用户明确要求（"存一下"或给路径）时才写文件 |
| 实现路线 | 纯指令型（A）：只用原生工具 WebFetch / Read / Bash / Write，零外部依赖 |

## 3. 架构

纯指令型 skill，单文件为主：

```
article-summary/
  SKILL.md                  # 触发描述 + 完整工作流 + 精简模板
  references/
    summary-template.md     # 完整输出模板（细节，按需加载）
```

- **名称**：skill `name` 与目录都用 `article-summary`（连字符）。当前目录是 `article_summary`（下划线），实现时重命名为 `article-summary`，与 Claude Code 按目录名加载的惯例一致。
- **零依赖**：不写 Python 脚本，不装库。利用 `Read` 原生读 PDF、`WebFetch` 读网页全文。

## 4. 详细设计

### 4.1 SKILL.md frontmatter

```yaml
---
name: article-summary
description: 总结天文/学术论文为中文结构化笔记。当用户给出 arXiv 链接或 ID、论文 PDF、LaTeX/文本文件、DOI 或期刊/ADS 链接，并希望得到结构化中文总结时使用。
---
```

description 是触发关键，需覆盖：arXiv、PDF、论文、总结、天文等词。

### 4.2 输入获取逻辑（四类）

1. **arXiv ID / 链接**
   - 从输入中规范化出 arXiv ID（支持 `2401.12345`、`arxiv.org/abs/...`、`arxiv.org/pdf/...` 等形式）。
   - 优先 `WebFetch` 抓 ar5iv 全文：`https://ar5iv.labs.arxiv.org/html/<id>`（HTML 全文，含公式、图表说明）。
   - ar5iv 抓不到 → `Bash` 用 curl 把 PDF 下到系统临时目录（如 `/tmp`）：`https://arxiv.org/pdf/<id>`，再用 `Read` 读取。

2. **本地 PDF**
   - 直接 `Read`。论文 > ~20 页时按段落分批读（先 摘要+引言，再 方法，再 结果，再 结论），用 `pages` 参数控制。

3. **本地 LaTeX / 文本**
   - 直接 `Read`，公式与结构原样保留。

4. **DOI / 期刊 / ADS**
   - `WebFetch` 抓落地页拿标题、作者、摘要、年份。
   - 若正文被付费墙挡住：用 `WebSearch` 或 arXiv 按标题搜预印本，找到后走 arXiv 流程拿全文。
   - 若全文始终拿不到：**仅基于摘要总结**，并在总结顶部明确标注 `⚠️ 仅基于摘要`。

### 4.3 输出模板

```markdown
# <论文标题>
**作者 · 年份 · 期刊/arXiv ID**

## TL;DR
<一两句话：做了什么 + 最重要的发现>

## 结构化拆解
- **背景与动机**：…
- **观测 / 方法**：…
- **数据样本**：…
- **主要结果**：…
- **科学意义**：…

## 公式与图表解读
- **关键公式**：<公式> — 各物理量含义
- **主要图表**：Fig X 表明…

## 局限 · 未来 · 相关工作
- **局限**：…
- **未来工作**：…
- **相关文献**：…
```

完整版（含每个字段的填写指引、示例）放 `references/summary-template.md`；SKILL.md 内嵌精简版。

### 4.4 语言规则

- 正文用中文。
- 保留英文：专业术语（redshift、metallicity）、天体名（M31、NGC 1068）、巡天/仪器名（JWST、SDSS、ALMA）、物理量与单位、作者名。
- 关键术语首次出现可加中文注，如 "光度函数 (luminosity function)"。

### 4.5 存档行为

- 默认：只在对话中展示总结。
- 触发存档：用户说"存一下/保存"或给出路径。
- 默认路径：当前目录下 `summaries/<arxiv-id 或 标题缩写>.md`。
- 写文件用 `Write`；文件名做安全化处理（去空格/特殊字符）。

### 4.6 错误处理

- ar5iv 无此论文 → 回退到 PDF 下载 + Read。
- 完全抓不到内容 → 明确告知用户来源拿不到，**不编造**。
- 只有摘要 → 总结顶部标注 `⚠️ 仅基于摘要`，并跳过无法填写的块或注明"摘要未涉及"。
- PDF 解析为乱码/扫描件 → 提示用户该 PDF 可能是扫描图片，建议提供 arXiv ID 或文本版。

## 5. 验证方式

- 用一个真实天文 arXiv 论文 ID 端到端跑一遍。
- 检查：模板四块齐全、中文质量、英文术语正确保留、未编造、按需存档能正确生成文件。
- 测一个本地 PDF 输入。
- 测一个只有摘要可得的 DOI，确认"仅摘要"标注生效。

## 6. 不做（YAGNI）

- 不做 Python 脚本 / 外部依赖。
- 不做术语词典文件（中文为主+保留英文，无需大词典）。
- 不做批量总结、不做 RSS 订阅、不做引用图谱——超出当前需求。
- 不做完全通用领域优化（聚焦天文/学术）。
